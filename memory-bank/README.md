# Memory Bank

This directory contains project knowledge, architecture decisions, and implementation notes for the Vocabulary Essay Analyzer PoC.

## Structure

- `architecture.md` - System architecture and design decisions
- `requirements.md` - Product requirements and specifications
- `decisions.md` - Key technical decisions and rationale
- `implementation-notes.md` - Implementation details and gotchas
- `api-spec.md` - API endpoint specifications
- `data-models.md` - Data schemas and models
- `task-list.md` - Development task list organized by epic
- `progress.md` - Project progress tracking and epic completion status
- `epic2-summary.md` - Detailed summary of Epic 2 implementation
- `epic6_8_tasks.md` - Detailed task breakdown for Epics 6-8
- `frontend-prd.md` - Frontend PRD for TanStack Router + Bun version

## Purpose

The memory bank serves as a single source of truth for:
- Why certain decisions were made
- How the system is structured
- What requirements must be met
- Important implementation details

This helps maintain context across development sessions and onboarding new contributors.
